## 1. 개요

현재 bubuncare.com은 로그인 후 대시보드만 존재합니다.
비로그인 방문자(자녀, 잠재 구독자)를 위한 **메인 홈페이지(/)** 를 신규 구성합니다.

**페이지 목적**
- 서비스 개념 설명 → 회원가입 CTA 전환
- 케어파트너 지원자 유입 경로 제공
- 지역 요소 없음 — 전국 서비스 포지셔닝

### 3-4. 이용 방법 섹션

**섹션 제목:** 3단계로 시작합니다

**스텝 3개:**

1. 부모님 프로필 생성
   - 자녀가 앱에서 부모님 기본 정보를 등록합니다. 5분이면 충분합니다.

2. 병원 카드 + 복약 정보 등록
   - 다니는 병원, 병명, 복용 중인 약을 병원별로 연결해 등록합니다. 주의 메모까지 기록하면 케어파트너가 현장에서 정확히 확인합니다.

3. 케어파트너 방문 + 리포트 수신
   - 주 2~3회 방문 후 자동 리포트 수신. 이상징후 발생 시 즉시 알림으로 연락됩니다.

### 3-5. 차별점 섹션

**섹션 제목:** 기존 앱과 무엇이 다른가요

**서브:** 해외 경쟁 서비스(Medisafe, CareZone)는 모두 노인이 직접 앱을 써야 합니다.

**2열 비교 카드:**

| bubuncare (강조 컬러) | 기존 앱들 (중립 컬러) |
|----------------------|---------------------|
| 케어파트너가 현장 직접 확인 | 노인 본인이 직접 체크 |
| 노인 앱 불필요 | 스마트폰 사용 필수 |
| 병원↔약 연결 관리 | 약 단독 관리 |
| 방문케어 + 데이터 통합 | 앱 알림만 제공 |
| 자녀가 모든 것을 등록·관리 | 방문케어 연동 없음 |

## 4. 디자인 가이드

**컬러 포인트:** #1D9E75 (그린 — 케어·건강·신뢰 연상)
**배경:** 흰색 기반, 섹션 구분은 border-bottom 0.5px
**폰트:** Noto Sans KR (한국어 가독성)
**카드 스타일:** 흰 배경 + 0.5px border + border-radius 12px
**요금제 추천 카드:** border 2px solid #1D9E75 (유일한 강조 예외)
**버튼:**
- Primary: background #1D9E75, color white
- Secondary: transparent + border 0.5px

**모바일 대응 (필수):**
- 기능 카드: 3열 → 1열
- 요금제 카드: 3열 → 1열 (Standard 먼저)
- 네비: 햄버거 메뉴

## 7. 라우팅 미들웨어 수정

```typescript
// middleware.ts
export function middleware(request: NextRequest) {
  const session = request.cookies.get('session')
  const isRoot = request.nextUrl.pathname === '/'

  if (isRoot && session) {
    return NextResponse.redirect(new URL('/dashboard', request.url))
  }
}
```
