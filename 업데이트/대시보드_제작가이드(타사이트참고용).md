## 1. 핵심 설계 철학

> **"좌열은 맥락, 우열은 내용"**

| 원칙 | 설명 |
|---|---|
| **Content-First** | 데이터와 행동 중심. 장식적 요소는 최소화 |
| **2-Panel Split** | 모든 페이지는 좌(컨텍스트) / 우(콘텐츠) 2분할로 구성 |
| **High Density** | 한 화면에서 최대한 많은 맥락을 제공 |
| **Minimalist** | 흑백 기반 그레이스케일. 컬러 포인트는 상태 표시 시에만 |
| **Sticky Left** | 좌열은 스크롤에 고정 → 조작(필터·탭·선택)은 항상 접근 가능 |

## 2. 레이아웃 구조 표준

### 2-1. 전체 페이지 래퍼

```jsx
<div className="bg-[#fafafa] min-h-screen font-sans text-slate-900">
  <div className="p-4 md:p-8 lg:p-10 mx-auto max-w-[1200px]">
    <div className="pt-8 grid grid-cols-1 lg:grid-cols-5 gap-8 items-start">
      {/* 좌열 col-span-2 */}
      {/* 우열 col-span-3 */}
    </div>
  </div>
</div>
```

### 2-2. 좌열 (40% — `col-span-2`)

```jsx
<div className="lg:col-span-2 space-y-5 lg:sticky lg:top-10">
  {/* 1. 페이지 제목 */}
  {/* 2. 요약 3카드 */}
  {/* 3. 주요 액션 버튼 (선택) */}
  {/* 4. 검색 / 필터 (선택) */}
  {/* 5. 수직 탭 or 목록 네비게이션 */}
</div>
```

### 2-3. 우열 (60% — `col-span-3`)

```jsx
<div className="lg:col-span-3 min-h-[60vh] pb-16">
  {/* 선택된 탭 or 항목 상세 콘텐츠 */}
</div>
```

### 2-4. 이용자 대시보드 (50/50 균등 분할)

```jsx
<div className="pt-6 grid grid-cols-1 lg:grid-cols-2 gap-8">
  <div>{/* 좌: 인사이트 카드, 요약 정보 */}</div>
  <div>{/* 우: 전체 응답 이력 */}</div>
</div>
```

| **Secondary Text** | `text-slate-400` | 레이블, 설명 |
| **Muted Text** | `text-slate-300` | 서브 정보, ID |
| **Border** | `border-slate-100` | 기본 카드 테두리 |
| **Border Hover** | `hover:border-slate-300` | 호버 강조 |
| **Active / CTA** | `bg-slate-900 text-white` | 선택된 탭, 액션 버튼 |

> - 삭제 위험 액션: `hover:text-red-400`

## 4. CSS 세부 규칙 (놓치기 쉬운 항목)

### 4-1. 폰트 패밀리

```js
// tailwind.config.js
theme: {
  extend: {
    fontFamily: {
      sans: ['ui-sans-serif', 'system-ui', 'sans-serif'],
    },
  },
}
```

```jsx
// 모든 페이지 최상위 래퍼에 반드시 적용
<div className="font-sans">
```

> [!WARNING]
> `font-sans`를 최상위에 선언하지 않으면 브라우저 기본 serif 폰트가 적용됩니다.
> 각 컴포넌트 파일에도 최상위 `<div>`에 `font-sans`를 명시하세요.

| 용도 | 클래스 | 주의 |
|---|---|---|
| 제목, 주요 수치, 활성 탭 | `font-black` (900) | 강한 강조가 필요한 핵심 요소 |
| 섹션 레이블, 주요 버튼 | `font-extrabold` (800) | 상위 위계의 구분선 역할 |
| 본문, 카드 내부 강조 | `font-bold` (700) | 표준 텍스트 레이어 |
| **보조 레이블, 진행률, 선택지** | `font-semibold` (600) | **v2.1 추가** — 가독성 우선 구역 |
| **상세 설명, 메타 정보** | `font-medium` (500) | **v2.1 추가** — 시각적 피로도 감소용 |
| `font-normal`, `font-light` | ❌ 사용 지양 | 기본 뼈대의 힘이 빠져 보임 |

### 4-2. 카드 라운딩 (Border Radius)

| 상황 | 클래스 | px | 사용처 |
|---|---|---|---|
| **카드, 패널** | `rounded-xl` | 12px | 모든 카드, nav, 입력창, 버튼 |
| **아이콘 컨테이너** | `rounded-lg` | 8px | `w-7 h-7` 아이콘 박스 |
| **뱃지, 태그, 토글** | `rounded-full` | 50% | pill 형태 요소 |
| **큰 아이콘 컨테이너** | `rounded-2xl` | 16px | 빈 상태 아이콘 박스 (`w-16 h-16`) |

> [!CAUTION]
> `rounded-md`(6px), `rounded-sm`(4px), `rounded-lg`(8px)를 카드에 쓰면 디자인이 깨집니다.
> 카드는 반드시 `rounded-xl`만 사용하세요.

```jsx
// ✅ 올바른 카드
<div className="bg-white p-5 rounded-xl border border-slate-100 shadow-sm">

// ❌ 잘못된 카드
<div className="bg-white p-5 rounded-lg border border-slate-200 shadow">
```

### 4-3. 그림자 (Box Shadow)

| 클래스 | 용도 |
|---|---|
| `shadow-sm` | 카드 기본 그림자 — **표준** |
| `shadow-md` | 버튼, 강조 카드 |
| `shadow-xl` | 모달, 아이콘 강조 (`w-11 h-11` 등) |
| `shadow-2xl` | 모달 컨테이너 |
| `shadow-inner` | 토글 트랙, 바 차트 배경 |

> [!WARNING]
> `shadow` (shadow-md 이상)를 카드에 기본으로 쓰면 과도하게 무거워 보입니다.
> 카드는 `shadow-sm`이 표준, 클릭 가능한 버튼만 `shadow-md` 이상 허용.

### 4-4. 테두리 (Border)

```jsx
// 카드 기본 테두리 — 반드시 이 조합
className="border border-slate-100"

// 호버 시 강조 테두리 — transition과 항상 세트
className="border border-slate-100 hover:border-slate-300 transition-all"

// 활성 선택된 항목 강조 (호버 아닌 클릭 후 고정)
className="border border-slate-900"

// 섹션 구분선 (페이지 제목 하단, 카드 내부 구분)
className="border-b border-slate-200"   // 제목 하단
className="border-b border-slate-50"    // 카드 내부 구분선

// 포커스 링
className="focus:ring-2 focus:ring-slate-900 focus:ring-offset-0 outline-none"
```

### 4-5. 타이포그래피 (전체 규격표)

| 요소 | 클래스 조합 |
|---|---|
| 수퍼 라벨 | `text-[12px] font-black text-slate-400 uppercase tracking-[0.4em] leading-none` |
| 페이지 제목 H1 | `text-2xl font-black tracking-tighter uppercase text-slate-900 leading-none` |
| 섹션 타이틀 H2 | `text-[14px] font-black text-slate-900 uppercase tracking-widest` |
| 소섹션 타이틀 H3 | `text-[13px] font-black text-slate-900 uppercase tracking-widest` |
| 카드 레이블 | `text-[9px] font-black text-slate-400 uppercase tracking-widest leading-none` |
| 카드 수치 | `text-base font-black text-slate-900 tracking-tighter leading-none` |
| 대형 수치 | `text-2xl font-black text-slate-900 tracking-tighter leading-none` |
| 본문 텍스트 | `text-[13px] font-bold text-slate-600 leading-relaxed` |
| 뱃지 / 태그 | `text-[10px] font-black uppercase tracking-widest` |
| 서브 메모 | `text-[10px] font-black text-slate-300 uppercase tracking-widest` |
| 입력 필드 텍스트 | `text-[13px] font-bold` |
| 버튼 텍스트 | `text-[13px] font-black uppercase tracking-widest` |

**letter-spacing 기준:**

| 클래스 | 값 | 사용처 |
|---|---|---|
| `tracking-[0.4em]` | 0.4em | 수퍼 라벨 (가장 넓음) |
| `tracking-widest` | 0.1em | 섹션 제목, 레이블, 버튼 |
| `tracking-tighter` | -0.05em | 큰 수치, H1 제목 (숫자 밀착) |
| `tracking-tight` | -0.025em | 탭 항목명, 목록 이름 |

### 4-6. 입력 필드 (Input / Select)

```jsx
// 텍스트 입력 — 표준
<input
  className="w-full px-4 py-3 bg-slate-50 border border-slate-100 rounded-xl font-bold text-[13px] outline-none focus:ring-2 focus:ring-slate-900 transition-all placeholder:text-slate-300"
/>

// 검색창 (아이콘 포함)
<div className="relative">
  <SearchIcon className="absolute left-4 top-1/2 -translate-y-1/2 text-slate-300" size={16} />
  <input className="w-full pl-11 pr-4 py-3 bg-white border border-slate-100 rounded-xl font-bold text-[13px] outline-none focus:ring-2 focus:ring-slate-900 shadow-sm transition-all" />
</div>

// 셀렉트
<select className="w-full px-4 py-2.5 bg-slate-50 border-none rounded-xl font-black text-[12px] text-slate-700 outline-none">
```

**입력 필드 체크리스트:**
- [ ] `outline-none` — 브라우저 기본 아웃라인 제거
- [ ] `focus:ring-2 focus:ring-slate-900` — 커스텀 포커스링 적용
- [ ] `placeholder:text-slate-300` — 플레이스홀더 색상 지정
- [ ] `bg-slate-50` (카드 내부) 또는 `bg-white` (독립 검색창)

### 4-7. 트랜지션 & 애니메이션

```jsx
// 표준 트랜지션 — 모든 상태 변화에 적용
className="transition-all"

// 데이터 로딩 스피너
className="animate-spin"

// 차트 막대 등장 (느린 드라마틱 효과)
style={{ height: `${percent}%` }}
className="transition-all duration-700"  // 또는 duration-1000

// 토글 원 이동
className="transition-all duration-300"

// 배경 장식 아이콘 (배경 그래픽)
className="opacity-[0.03] group-hover:opacity-[0.08] transition-opacity"
```

**애니메이션 사용 기준:**

| 클래스 | 용도 | 주의 |
|---|---|---|
| `animate-spin` | 로딩 스피너 전용 | 스피너 한정 |
| `transition-all` | 모든 호버·클릭·상태 변화 | 빠뜨리면 딱딱하게 느껴짐 |
| `duration-300` | 토글, 버튼 배경색 변화 | |
| `duration-700` / `duration-1000` | 차트 막대 높이 변화 | 너무 빠르면 효과 반감 |

### 4-8. opacity 활용 패턴

| 클래스 | 용도 |
|---|---|
| `opacity-20` | 로딩 텍스트 ("데이터 동기화 중...") |
| `opacity-[0.03]` | 카드 배경 장식 아이콘 기본 |
| `opacity-[0.08]` | 카드 배경 장식 아이콘 호버 |
| `opacity-10` | 목업 내부 텍스트 줄 (스켈레톤 효과) |
| `text-white/10` 등 `/숫자` | 다크 배경 위 반투명 요소 |
| `bg-white/10` | 다크 헤더 내부 배지 배경 |

```jsx
// 예시: 다크 헤더 내 반투명 뱃지
<div className="px-4 py-1 bg-white/10 rounded-full text-[11px] font-black backdrop-blur-sm">
  Top Segment: ENFP
</div>
```

### 4-9. z-index 컨벤션

| 값 | 클래스 | 용도 |
|---|---|---|
| 10 | `z-10` | 카드 내부 relative 요소 |
| 100 | `z-[100]` | 모달 오버레이 |
| (미지정) | — | 일반 카드, 페이지 요소 |

```jsx
// 모달 오버레이 — 반드시 z-[100]
<div className="fixed inset-0 z-[100] bg-slate-900/50 backdrop-blur-sm">
```

### 4-10. overflow 처리 패턴

| 상황 | 클래스 |
|---|---|
| 카드 내부 배경 아이콘 넘침 방지 | `relative overflow-hidden` |
| 긴 텍스트 한 줄 처리 | `truncate` (또는 `overflow-hidden text-ellipsis whitespace-nowrap`) |
| 좌열 목록 스크롤 제한 | `max-h-[50vh] overflow-y-auto` |
| 테이블 가로 스크롤 | `overflow-x-auto` |
| 모달 내용 넘침 방지 | `overflow-hidden` (모달 컨테이너에) |

### 4-11. 장식 아이콘 (Decorative Icon) 패턴

카드 우측 하단에 배치하는 배경 효과:

```jsx
<div className="... relative overflow-hidden group">
  {/* 콘텐츠 */}
  
  {/* 배경 장식 — 반드시 absolute, 마지막 자식 */}
  <IconName
    className="absolute -bottom-6 -right-6 text-white/5 w-32 h-32"
  />
  {/* 또는 카드 우상단 */}
  <div className="absolute top-0 right-0 p-8 opacity-[0.03] group-hover:opacity-[0.08] transition-opacity">
    <Target size={120} />
  </div>
</div>
```

### 4-12. group / group-hover 패턴

카드 전체에 hover 효과를 아이콘까지 동시 적용:

```jsx
<div className="... group">
  <div className="... group-hover:bg-slate-900 group-hover:text-white transition-all">
    <Icon size={14} />
  </div>
  <span className="... group-hover:text-slate-900">텍스트</span>
</div>
```

## 9. 피해야 할 패턴 (Anti-Patterns)

| ❌ 금지 | ✅ 대신 |
|---|---|
| `font-normal`, `font-light` | `font-black` ~ `font-medium` 위계 활용 |
| `rounded-md`, `rounded-lg` 카드 | `rounded-xl` 통일 |
| `shadow` 이상을 카드 기본값으로 | `shadow-sm` 표준 |
| `text-sm`, `text-lg` 등 Tailwind 기본 사이즈 | `text-[13px]` 등 px 명시 |
| 임의 색상 (blue, green, purple 등) | slate 그레이스케일 + 위험/피크 포인트만 |
| border 없는 카드 | `border border-slate-100` 필수 |
| `outline` 제거 후 focus 표시 없음 | `outline-none` + `focus:ring-2` 세트 |
| 카드를 별도 페이지로 라우팅 | 우열 인라인 상세 표시 |
| `footer` 대규모 사용 | 우열 `pb-16` 으로 충분 |
| 4개 이상 컬럼 요약 카드 | 반드시 3개 유지 |
