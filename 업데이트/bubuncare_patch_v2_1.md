## 패치 1 — 온보딩 첫 화면

### 대상 페이지
`/` 또는 `/dashboard` — 로그인 후 care_profiles가 0개일 때

### 조건
```
care_profiles 테이블에서 child_user_id = auth.uid() 인 row가 없을 경우
→ 온보딩 화면 표시
→ 있을 경우 → 기존 대시보드 표시
```

### 화면 구성
```
[큰 타이틀]
부모님 건강 정보를
한눈에 관리하세요

[3단계 안내 — 아이콘 + 텍스트]
1. 병원 카드 등록   →   2. 복약 정보 등록   →   3. 대시보드 확인

[CTA 버튼]
"부모님 프로필 만들기" → /profile/new 이동
```

### 주의사항
- 프로필이 이미 있는 재방문 사용자에게는 절대 표시 안 됨
- 로그인 직후 1회만 표시 (이후 대시보드 직행)

## 패치 2 — 빈 상태(Empty State) 3곳

### 2-1. 병원 카드 없을 때 (/profile/[id]/hospitals)

```
[아이콘: 병원 심볼]
아직 등록된 병원이 없어요

부모님이 다니시는 병원을 먼저 등록하면
복약 정보와 자동으로 연결됩니다

[버튼] "+ 첫 번째 병원 등록하기" → /profile/[id]/hospitals/new
```

### 2-2. 복약 없을 때 (/profile/[id]/medications)

```
[아이콘: 알약 심볼]
아직 등록된 약이 없어요

※ 병원 카드를 먼저 등록하면
   약과 병원을 연결해서 관리할 수 있어요

[버튼 1] "+ 약 등록하기" → /profile/[id]/medications/new
[버튼 2, 병원카드 0개일 경우만 표시] "병원 먼저 등록하기" → /profile/[id]/hospitals/new
```

### 2-3. 대시보드 — 약/병원 모두 없을 때 (/dashboard)

```
[카드 형태로 표시]
등록된 정보가 없습니다
부모님 프로필에서 병원과 복약 정보를 등록해 주세요

[버튼] "지금 등록하기" → /profile/[id]/hospitals/new
```

## 패치 3 — 병원 미등록 상태에서 복약 등록 시도 시 유도

### 대상 페이지
`/profile/[id]/medications/new`

### 조건
```
해당 profile_id 기준 hospital_cards 테이블 row가 0개일 경우
→ 약 등록 폼 상단에 안내 배너 표시
```

### 배너 UI
```
[주황색 안내 배너 — 폼 최상단]
병원 카드를 먼저 등록하면 약과 병원을 연결할 수 있어요
[링크] 병원 등록하러 가기 →
```

### 주의사항
- 배너는 안내일 뿐 — 약 등록 자체를 막으면 안 됨
- hospital_id 필드는 선택(optional) 유지
- 배너 우측에 X 닫기 버튼 제공 (세션 내 1회 닫으면 재표시 안 함)
